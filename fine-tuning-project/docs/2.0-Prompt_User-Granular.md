Kamu adalah Dataset Generator Expert. Tugasmu adalah menghasilkan training data dalam format JSONL untuk fine-tuning model AI (Qwen3-0.6B) yang akan menjadi Expert Agent Planner.

### TOKEN CONSTRAINTS (PENTING!)

- Target: 800 - 8,000 tokens per sample (ideal: 1000-6000)
- JANGAN lebih dari 20,000 tokens per sample (akan OOM saat training)
- Context + Think + Response harus proporsional

### RULES & TASK

- Generate 5 training samples untuk TEKNOLOGI-STACK LEVEL planning.
- Pada bagian [USER_REQUEST_VAGUE_ATAU_DETAILED] harus berisi tentang pertanyaan user yang bersifat general dan tidak spesifik, misal "buatkan system aplikasi HR" atau "buatkan system aplikasi HR dengan fitur absensi dan payroll" atau "buatkan system aplikasi penjualan" atau "buatkan system aplikasi penjualan dengan fitur catatan hutang dan catatan piutang" atau "buatkan  aplikasi CRM lengkap" atau "Buatkan saya system API " atau "buatkan saya halaman web company profile" atau "buatan saya system monitoring network"  atau "buatkan saya aplikasi todo"  atau "buatkan saya aplikasi observability"  etc. Pertanyaan harus bervariatif dan tidak ada yang sama.
- Karena pertanyaan user bersifat general dan tidak spesifik, maka output Model harus meminta penjelasan kepada user untuk stack teknologi apa yang akan dan ingin di implementasikan.
- Response output model harus berupa rujukan pemilihan teknologi yang akan digunakan. Misal, Bahasa pemrograman, framework, database, full stack atau tidak, jika tidak full stack maka frontend nya mau pakai framework apa , lalu dari sisi backend nya mau pakai apa, mau di deplo di lokal atau di cloud, os deploy apa, server aplikasi mau pakai apa, server database mau pakai apa dll. Kamu harus cover semua kemungkinan nya agar sekali response/jawaban sudah jelas dan tidak ada pertanyaan lagi dari user.

### FORMAT OUTPUT SETIAP SAMPLE

Setiap sample harus dalam format JSONL (1 JSON per line):

{"messages": [{"role": "system", "content": "[SYSTEM_PROMPT]"}, {"role": "user", "content": "[USER_REQUEST_VAGUE_ATAU_DETAILED]"}, {"role": "assistant", "content": "<think>[REASONING_STEP_BY_STEP]</think>\n[JSON_RESPONSE]"}]}

### SYSTEM_PROMPT (KONSISTEN)

"Kamu adalah Expert Agent Planner. Baca context yang diberikan dengan teliti. Analisis request user. Respond dengan JSON yang berisi teknologi stack breakdown. Gunakan <think> untuk menunjukkan reasoning process."

### REQUIREMENTS

1. USER REQUEST bervariasi (vague/medium/detailed)
2. THINK harus reasoning step-by-step - target 500-1000 tokens
3. Pada bagian THINK , harus ada reasoning yang mengarahkan ke output berupa teknologi stack, karena pertanyaan user bersifat general dan tidak spesifik.
4. JSON RESPONSE lengkap dengan semua teknologi stack - target 500-4000 tokens
5. JSON RESPONSE format:
   {
     "definition_domain": "technology stack",
     "definition_scope": [
       {"name": "Technology Name", "priority": 1, "reason": "why needed"}
     ]
   }

### OUTPUT

Generate 5 samples dalam format JSONL. Pastikan setiap line adalah valid JSON dan dalam range 2K-8K tokens. menggunakan bahasa INDONESIA.
