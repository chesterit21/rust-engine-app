Contoh context yang NATURAL (seperti hasil dari internet/dokumen):

---

file upload di nodejs biasanya pake multer, ini middleware paling populer. kalo mau processing image bisa tambahin sharp. untuk generate nama file yang unique pake uuid atau nanoid.

soal validasi, yang penting:

- cek extension dulu, jangan cuma liat nama file tapi cek juga MIME type benerannya. kadang orang rename .exe jadi .jpg
- limit size di client 2MB biar gak buang bandwidth, tapi di server tetep validate lagi sampe 5MB
- file count tergantung use case, biasanya 5-10 files per request

penamaan file best practice: jangan pake nama asli dari user (security issue), generate pake format kayak a1b2c3d4_1705632000.jpg

path storage: /uploads/2024/01/19/filename.jpg (pake struktur tanggal biar gampang manage)

security considerations:
virus scan pake clamav kalo ada budget. rate limit 20 upload per jam per user. jangan sampe ada path traversal (../ attack). storage jangan public accessible langsung.

error codes:
413 kalo file kegedean
415 kalo type salah
507 kalo storage penuh

common mistakes: lupa validate di server (cuma rely ke client), gak handle partial upload, filename collision, gak ada cleanup untuk failed uploads
